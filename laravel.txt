// Laravel hoạt động như thế nào 
Đầu tiên request chạy vào public/index 
public/index vendor/autoload autoload các thư viện trong vendor 
public/index bootstrap/app khởi tạo một cái kế thừa của cái khung bootstrap laravel, load kernel lên để kiểm tra request, load exception để lên thông báo lỗi 
- Http/Kernel: 
	Khi request chạy vào Http/Kernel - cấu hình xử lý lỗi 
									 - cấu hình ghi log 
									 - load env 
									 - đăng ký thư viện cho middleware và route để lọc request 
- Service Provider / Service Container: 
	Khi request chạy vào Service Provider / Service Container - load Providers (register and boot các phương thức khai báo) 
															  - load Aliases (gọi static) 
- Route: Request chạy vào 
	Request được kiểm tra bởi middleware đã đăng ký trước đó 
- Controller nhận request
